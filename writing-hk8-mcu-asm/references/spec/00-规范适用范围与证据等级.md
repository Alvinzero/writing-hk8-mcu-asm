# 00 规范适用范围与证据等级

## 1. 目的

本规范把 `D:\hk64s8x-cli` 中散落的编译器行为、ASM 实验、工件、实板结果和失败假设统一成可复制的工程约束。目标不是描述所有可能的 HK64S8x 硅片，而是建立一个**可追溯、能被 AI/CI 执行、冲突时知道听谁的**基线。

## 2. 已覆盖范围

- 芯片家族：HK64S8x。
- 已明确验证型号：HK64S8101。
- 程序空间基线：1024 × 16-bit MTP word。
- 汇编输入：公司 HK_ASM_Compiler 语法、`REG825.INC` 命名。
- 工具链：公司 IDE、Python 源码模块 CLI、公司 simulator、在线烧录/回读。
- 外设专题：GPIO、bit-bang I2C、SSD1306 128×64、当前混合极性四位数码管。
- 样本：53 个 ASM、53 个 MAP、53 个 BIN、53 个 HEX、34 个 hkproj。

## 3. 不自动泛化的内容

下列结论默认只属于当前板级 profile，不能因芯片相同而跨板复制：

- SDA=PB7、SCL=PB6。
- SSD1306 7-bit 地址 0x3C / write byte 0x78。
- 四位数码管段线、COM 接线、混合极性、左到右位序和 `PA_PIO=60H` 全关值。
- NOP 数、延时循环与实际总线频率。
- OPTION、晶振、电压、上拉电阻、唤醒源。

更换芯片型号、编译器版本、板卡或外设模块时，必须建立新 profile 并重跑基准探针。

## 4. 规范词

| 词 | 含义 |
|---|---|
| MUST / 必须 | 违反即不合规；按 severity 决定是否阻断构建、烧录或交付 |
| SHOULD / 应 | 原则上遵守；偏离需要书面理由、影响分析和验证 |
| MAY / 可 | 可选实践 |

Severity：

| 级别 | 处置 |
|---|---|
| BLOCKER | 立即停止构建/烧录/交付 |
| ERROR | 修复后才能交付 |
| WARNING | 评审并记录豁免；默认不进入量产 |
| INFO | 提示或统计 |

机器定义见 `rules/asm-rules.json`。

## 5. 证据等级

| 等级 | 定义 | 典型例子 |
|---|---|---|
| E1 | 实板确认 | OLED 正确字符/头像、数码管正确位序、I2C ACK 修复 |
| E2 | 公司 IDE 真实构建产物 | MAP 地址、BIN/HEX 中 DB 物理编码 |
| E3 | 当前编译器源码 | DB 未生成、跳转超范围截断、simulator 固定 page 0 |
| E4 | 自动测试或独立编译探针 | 65 个指令变体代表机器字、JMP/CALL 数字目标失败 |
| E5 | 项目文档或源码注释 | README、SKILL、实验文件说明 |
| E6 | 合理推断 | 工程防御性约束、尚未实板覆盖的建议 |
| OPEN | 尚无足够证据 | ROM 总线原因、RET A,#K 硬件语义 |

## 6. 冲突优先级

    E1 实板输出
    > E2 公司 IDE 真实产物
    > E3 当前编译器源码
    > E4 自动测试/编译探针
    > simulator
    > E5 项目注释
    > E6 推断

“优先”不代表低等级资料无用。例如，IDE BIN 能证明物理编码，但当其表面排列与实板消费顺序冲突时，运行时规范服从 E1，同时保留 E2 作为工件审计规则。

## 7. 结论的状态

- **active**：可直接作为规则使用。
- **restricted**：只能在特定条件、实验或有额外证据时使用。
- **deprecated**：历史结论，禁止新代码采用。
- **open**：待确认，不能由 AI 自动补齐。

项目里 `hello_db_tabl_tabh_hxd_probe.asm` 的 DB 补偿假设属于反例；`ssd1306_oled_*_db_raw_verified.asm` 属于 active 基线。

## 8. 版本绑定

`analysis/source-manifest.json` 保存关键源码和工件 SHA256。本规范的结论绑定这些输入。以下变化必须触发规范重验证：

- 公司 IDE 或 Python 源码模块更新。
- `instruction_set.xlsx/json`、`register_set.xlsx/json`、`REG825.INC` 更新。
- 芯片型号或 revision 变化。
- 烧录协议、BIN 生成、模拟器模型变化。
- 板级接线、电压、主频或外设模块变化。

## 9. 规则例外

例外申请至少包含：

1. 规则 ID。
2. 偏离理由。
3. 影响范围和失败模式。
4. 替代控制措施。
5. E1/E2 级验证计划与结果。
6. 有效期限、芯片/板卡/工具链版本。

BLOCKER 规则不得仅凭代码评审口头豁免；必须新增更高或同等级证据并升级规范版本。
