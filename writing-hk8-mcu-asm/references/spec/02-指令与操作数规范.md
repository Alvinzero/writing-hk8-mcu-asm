# 02 指令与操作数规范

## 1. 本文档的边界

本项目使用 2026-07 公司指令元数据整理出 65 个指令变体、56 个 mnemonic，并对每个变体生成了一个代表性源码模块编译探针。当前结果为 **65/65 能生成与元数据预期一致的 16-bit machine word**。原始结构化快照见 [rules/instruction-metadata.json](rules/instruction-metadata.json)，规范化结果见 [rules/instruction-reference.json](rules/instruction-reference.json) 和 [analysis/probe-results.json](analysis/probe-results.json)。

必须正确理解该结论：

- 它证明的是“当前 Python assembler 模块能识别代表写法并生成目标机器字”。
- 它不等于 65 个变体都已在 HK64S825 实板上完成语义、flags、边界值和时序验证。
- `TABL/TABH` 的项目内使用路径有 E1 实板证据；`RET A,#K`、`CPL/CPLR` 仍有语义 OPEN；`IDLE/STOP` 需要具体板级低功耗测试。
- AI 不得把 `compile_probe.status=passed` 翻译成“硬件功能已验证”。

适用规则：`HK-SYN-002..011`、`HK-LAYOUT-006`、`HK-TABLE-004..005`。

## 2. 操作数模型

| 符号 | 工程含义 | 合法范围 | 推荐写法 | 审查重点 |
|---|---|---:|---|---|
| `A` | 累加器，很多指令和子程序的隐式共享资源 | 芯片定义 | `MOV A,80H` | 默认被调用者可破坏 |
| `R` | 8-bit 数据空间操作数；可为正式 SFR 名或直接地址 | `0x00..0xBF`，其中空间含义不同 | `MOV PB_PIO,A`、`MOV 90H,A` | 区分 SFR 与 SRAM |
| `#K` | 8-bit immediate | `0..255` | `MOV A,#80H` | 不得溢出或混搭进制后缀 |
| `b` | bit index | `0..7` | `BCLR PB_POE,7` | 位号必须与 board profile 一致 |
| jump `K` | 10-bit 程序 word 地址 | `0..1023` | `JMP MAIN_LOOP` | 只能使用标签或 `EQU` 符号 |

`R` 是编码字段，不表示所有地址都具有相同生命周期。`00H..7FH` 是 SFR，`80H..BFH` 是 SRAM；OPTION 是独立配置空间，不能作为普通 `R` 使用。

## 3. 数值语法

允许且已通过代表探针的形式：

```asm
MOV A,#0x80
MOV A,#80H
MOV A,#128
MOV 80H,A
ORG 0x0100
TABLE_END EQU 03FFH
```

禁止混合两种十六进制标记：

```asm
MOV A,#0x80H          ; 错误：0x 与 H 混搭
```

统一约定：

- 协议字节、位掩码、SFR/SRAM 地址优先使用十六进制。
- 人类可读数量可使用十进制，但应通过有语义的 `EQU` 名称表达。
- 立即数前的 `#` 属于操作数语法，不是数值本身的一部分。
- 不允许依赖编译器对溢出值的截断；k8、bit、k10 在进入编译器前就必须通过静态检查。

## 4. 规范化拼写

原始元数据存在已确认的脏字段，交付源码必须使用右侧形式：

| 原始字段问题 | 规范形式 | 处理 |
|---|---|---|
| `MOV A,#K` 的 operand type 被写成 `r8` | `MOV A,#K`，按 k8 检查 | 元数据修正 |
| `XOR A.#K` 使用句点 | `XOR A,#K` | 禁止复制句点写法 |
| 历史导出的 `BTSZ,R,b` | `BTSZ R,b` | 2026-07 元数据已修复，reference 继续兼容旧输入 |
| 历史导出的 `BTSNZ,R,b` | `BTSNZ R,b` | 2026-07 元数据已修复，reference 继续兼容旧输入 |

指令拼写以 `rules/instruction-reference.json[].asm_syntax` 为唯一机器参考，不以历史 Excel 单元格或探针文件中的偶然写法为准。

## 5. 指令族使用约束

### 5.1 算术、逻辑和写回变体

`ADD/SUB/AND/OR/XOR` 同时存在 immediate 或 `R` 形式；带 `R` 后缀的变体常表示写回数据空间的不同编码。由于当前证据主要是编译探针，调用者必须按芯片正式资料确认：

- 结果写入 A 还是写回 `R`。
- carry、zero 等 flags 是否被改变。
- rotate/shift 是否经过 carry。

不得仅凭 mnemonic 的英文习惯推断 `ADDR/ADDCR/SUBR/SUBCR/ANDR/ORR/XORR` 的全部硬件语义。关键算法需要最小实板真值表。

### 5.2 `INC/DEC` 与 skip 变体

`INCSZ/INCSZR/DECSZ/DECSZR` 等属于“修改并按结果跳过下一条”的紧耦合控制流。规范：

```asm
DECSZR 81H
JMP SEND_LOOP          ; 被 skip 的必须是唯一、明确的一条机器指令
```

- skip 指令后的语句必须是一条真实机器指令，不得假设伪指令或宏只占一个 word。
- 代码评审必须同时看 skip 指令和紧随其后的下一条。
- 对这两行之间的插入、重排或自动格式化视为控制流修改，必须重新测试。

### 5.3 比较 `SE` 和 A 重载

项目历史故障表明，连续比较不能依赖上一次 `SE` 后的 A/flags 状态。保存原值，并在每次比较前重载：

```asm
MOV 97H,A

MOV A,97H
SE #1
JMP VALUE_NOT_1
; ...

VALUE_NOT_1:
MOV A,97H
SE #2
JMP VALUE_NOT_2
```

不合规形式：

```asm
SE #1
JMP VALUE_NOT_1
SE #2                 ; 未重新装载 A
```

### 5.4 bit 指令

`BSET/BCLR/BCPL/BTSZ/BTSNZ` 的 `b` 必须在 `0..7`。用于 GPIO 时还必须能追溯到 board profile：

```asm
BCLR PB_POE,7         ; PB7=SDA；释放输出使能
```

`BCPL` 的编码探针已通过，但对会造成总线争用、位选闪烁或安全输出变化的场景，不建议用 toggle 代替显式 `BSET/BCLR`。

## 6. `JMP/CALL` 目标规范

当前源码模块的直接数字目标存在 token 类型兼容问题，以下代表探针失败：

```asm
JMP 03FFH
CALL 03FFH
JMP 0x03FF
CALL 0x03FF
```

诊断为：`期望操作数格式: [K], 实际: [R]`。

必须改为：

```asm
JMP MAIN_LOOP

ROM_LAST EQU 03FFH
JMP ROM_LAST
```

额外门禁：

- 目标必须在 `0x0000..0x03FF` word 范围内。
- 编译器对超范围目标只 warning 后执行 `& 0x3FF` 截断的路径，交付流程必须把该 warning 提升为 BLOCKER。
- `CALL` 深度和硬件栈容量尚无本规范基线资料，设计应减少嵌套，并把最大调用深度列为 review 项。

## 7. 返回、中断和看门狗

| 指令 | 交付策略 | 原因 |
|---|---|---|
| `RET` | 可用 | 代表编码探针通过；仍需遵守子程序契约 |
| `RETI` | 可用于已确认中断向量 | 中断现场和标志恢复仍需芯片资料/实板验证 |
| `RET A,#K` | **正式关键路径禁用** | 可生成 `0xA1KK`，但硬件语义未确认 |
| `CLRWDT` | 可用但受 OPTION/WDT 配置约束 | 不能用它掩盖失控循环或错误时序 |

任何中断服务程序必须声明 A、flags、共享 SRAM 的保存策略；默认普通子程序不可重入。

## 8. 受限指令

### `CPL/CPLR`

当前说明材料对确切按位语义、写回目标存在疑点。规范状态为 restricted：

- 在量产关键逻辑中禁用。
- 若确需采用，先对 `00H`、`01H`、`55H`、`AAH`、`FFH` 等输入做实板真值表，并记录输出和 flags。

### `IDLE/STOP`

编码通过不代表板级可用。必须分别验证：

- 唤醒源、唤醒延迟、复位原因。
- WDT 在低功耗状态下的行为。
- GPIO 输出保持、上拉/开漏状态。
- I2C/OLED/数码管离开和恢复时的电气状态。
- 不同 OPTION、主频和供电条件。

在上述证据缺失时，不得把低功耗指令纳入 flash candidate。

## 9. `TABL/TABH`

规范只允许已验证的成对读取路径：

```asm
; DB B0,B1
MOV A,TABLE_INDEX
TABL
CALL CONSUME_BYTE

MOV A,TABLE_INDEX      ; 必须重装，TABL 已覆盖 A
TABH
CALL CONSUME_BYTE
```

同时满足：

```text
(table_word_address >> 8) == (TABL_or_TABH_instruction_address >> 8)
```

详细的 DB 物理编码、分页和 MAP 审计见 [04-程序布局-ORG-查表规范.md](04-程序布局-ORG-查表规范.md)。

## 10. 子程序调用契约

每个复用函数入口前至少声明：

```asm
; NAME:     I2C_SEND
; IN:       A = byte
; OUT:      80H bit7 = sampled ACK level
; CLOBBERS: A, 80H, 81H, flags
; REENTRANT:no
; TIMING:   board_profile.clock_hz + measured SCL waveform
```

调用者不得依赖未声明保存的 A、flags 或 SRAM。一个函数调用另一个函数时，scratch 分配必须覆盖整个嵌套生命周期，而不是只看单个函数内部。

## 11. 指令采用分级

| 级别 | 条件 | 允许状态 |
|---|---|---|
| L0 编码可生成 | 仅 compile probe passed | 可用于实验草案，不得宣称硬件语义完整 |
| L1 仿真覆盖 | simulator 有一致结果 | 仍不能覆盖 simulator 模型缺陷 |
| L2 工件审计 | 公司 IDE word 与 MAP 符合预期 | 可进入受控烧录候选 |
| L3 实板语义 | 边界值、flags、时序有 E1 记录 | 可成为公司模板路径 |

新指令进入公司关键模板前，至少应达到 L2；涉及外设、电源、查表或控制流的关键语义应达到 L3。

## 12. 审查清单

- [ ] 指令拼写来自修正版 reference。
- [ ] k8、bit、k10 全部在范围内。
- [ ] 无 `#0x..H` 混搭。
- [ ] `JMP/CALL` 使用标签或 `EQU` 符号。
- [ ] skip 指令及下一条作为一个单元评审。
- [ ] 连续 `SE` 前重新装载 A。
- [ ] 子程序声明 A、flags、SRAM clobbers。
- [ ] 未在关键路径使用 `RET A,#K`、`CPL/CPLR`。
- [ ] `IDLE/STOP` 有板级测试报告。
- [ ] `TABL/TABH` 遵守原始字节顺序、A 重装和同页规则。
