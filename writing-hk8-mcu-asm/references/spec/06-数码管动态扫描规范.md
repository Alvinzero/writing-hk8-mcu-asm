# 06 数码管动态扫描规范

## 1. 本文档只绑定当前开发板 profile

下列接线和极性来自项目实板，不得泛化到所有 HK64S825 或所有四位数码管：

### 段线

| GPIO | 段 |
|---|---|
| `PB7` | A |
| `PB6` | B |
| `PB5` | C |
| `PB4` | D |
| `PB3` | E |
| `PB2` | F |
| `PB1` | G |
| `PB0` | DP / SEGH |

### 位选

| GPIO | 逻辑名 | 极性 | 选通 | 关闭 |
|---|---|---|---|---|
| `PA2` | COM0 | 共阳 | high | low |
| `PA3` | COM1 | 共阳 | high | low |
| `PA5` | COM2 | 共阴 | low | high |
| `PA6` | COM3 | 共阴 | low | high |

物理从左到右：

```text
COM2, COM3, COM0, COM1
```

全部关闭：

```text
COM0=0, COM1=0, COM2=1, COM3=1
PA_PIO = 0x60
```

适用规则：`HK-7SEG-001..007`。

## 2. 初始化

初始化顺序固定为：目标 `POD` -> 安全 `PIO` -> 最后开启 `POE`。PA 只归本任务所有 `PA2/PA3/PA5/PA6`，因此必须 read-modify-write 保留非本任务位；PB0..PB7 在当前 profile 中全部属于段线，才允许整口写入。不得先打开 `POE` 再补写输出锁存值。

```asm
MOV A,PA_POD
AND A,#93H             ; 93H 是 6CH 的反掩码，只清 PA2/PA3/PA5/PA6，建立推挽输出
MOV PA_POD,A
MOV A,#00H
MOV PB_POD,A           ; PB0..PB7 均为本任务段线，建立推挽输出

MOV A,PA_PIO
AND A,#93H             ; 先清目标位，同时保留其他 PA 位
OR A,#60H              ; 预装全部 COM 关闭状态
MOV PA_PIO,A
MOV A,#FFH
MOV PB_PIO,A           ; 在输出使能前预装段线安全值

MOV A,PA_POE
OR A,#6CH
MOV PA_POE,A           ; 最后只开启目标位选输出
MOV A,#FFH
MOV PB_POE,A           ; 最后开启 PB0..PB7 段线输出
```

只有当前 PinContract 明确拥有整口时才允许整口写入。若与 I2C、按键或传感器共享 port，必须按 ownership 对 `POD/PIO/POE` 分别做 read-modify-write，保留非本任务位。

## 3. 两套段码

bit 顺序为 `PB7=A ... PB1=G, PB0=DP`。DP 关闭时：

| 数字 | 共阴：高亮 | 共阳：低亮 |
|---:|---:|---:|
| 0 | `FCH` | `03H` |
| 1 | `60H` | `9FH` |
| 2 | `DAH` | `25H` |
| 3 | `F2H` | `0DH` |
| 4 | `66H` | `99H` |
| 5 | `B6H` | `49H` |
| 6 | `BEH` | `41H` |
| 7 | `E0H` | `1FH` |
| 8 | `FEH` | `01H` |
| 9 | `F6H` | `09H` |

共阳表是当前 bit mapping 下共阴表的按位反相。DP：

- 共阴位点亮 DP：bit0=1。
- 共阳位点亮 DP：bit0=0。

不得把一套共阴表用于四个位；当前板的后两位是共阳。

## 4. 物理位序优先于逻辑编号

显示业务数组通常按视觉顺序保存：

```text
digit[0] digit[1] digit[2] digit[3]
left                                  right
```

当前板映射为：

| 视觉索引 | COM | 极性 | 示例 SRAM |
|---:|---|---|---|
| 0 | COM2 | 共阴 | `90H` 千位 |
| 1 | COM3 | 共阴 | `91H` 百位 |
| 2 | COM0 | 共阳 | `92H` 十位 |
| 3 | COM1 | 共阳 | `93H` 个位 |

固定 `1234` 显示成 `3412` 时，优先修正 COM 物理位序，不要先修改数字值或交换段码表。

## 5. 无鬼影扫描序列

每一位严格执行：

```text
全部 COM 关闭
-> 计算/写入当前位段码
-> 只打开当前 COM
-> 保持短延时
-> 全部 COM 关闭
```

示例：

```asm
; 左 1: COM2，共阴
MOV A,#60H
MOV PA_PIO,A           ; all off
MOV A,90H
CALL GET_CC_SEG
MOV PB_PIO,A
MOV A,#40H             ; COM2 low, others off
MOV PA_PIO,A
CALL SCAN_DELAY
MOV A,#60H
MOV PA_PIO,A
```

禁止在任意 COM 已打开时切换 `PB_PIO` 段码，否则会串影、短暂显示上一位数据，严重时表现为“全亮”。

## 6. 当前板四个位选值

在保持非目标位关闭的前提下：

| 视觉位置 | COM | `PA_PIO` |
|---:|---|---:|
| 全关 | none | `60H` |
| 左 1 | COM2 active low | `40H` |
| 左 2 | COM3 active low | `20H` |
| 左 3 | COM0 active high | `64H` |
| 左 4 | COM1 active high | `68H` |

这些值包含所有四个 COM 的完整状态；不要只 toggle 单个位后遗留上一位选通。

## 7. 状态与 scratch 分离

推荐项目分配示例：

| 地址 | 用途 | class |
|---:|---|---|
| `90H` | 千位 BCD | persistent |
| `91H` | 百位 BCD | persistent |
| `92H` | 十位 BCD | persistent |
| `93H` | 个位 BCD | persistent |
| `94H` | display hold count | scratch/loop state |
| `95H..96H` | scan delay | scratch |
| `97H` | segment conversion input | scratch |

禁止让 `90H..93H` 同时充当 delay counter、I2C shift、table index 或 conversion scratch。完整原则见 [03-寄存器与内存使用规范.md](03-寄存器与内存使用规范.md)。

## 8. 段码转换的 `SE` 规则

转换前保存原始数字，每次比较前重载 A：

```asm
GET_CC_SEG:
  MOV 97H,A

  MOV A,97H
  SE #0
  JMP CC_NOT_0
  MOV A,#FCH
  RET

CC_NOT_0:
  MOV A,97H
  SE #1
  JMP CC_NOT_1
  MOV A,#60H
  RET
```

连续 `SE` 不重载 A 是历史高风险模式，可能造成某些数字分支错乱或状态被误判。

## 9. BCD 计数

四位十进制状态应独立 carry：

```text
0001 -> 0002 -> ... -> 0009 -> 0010 -> ... -> 9999 -> 0001
```

每次进位流程：

1. 增加个位。
2. 与 `0AH` 比较；未到 10 立即返回。
3. 清个位，增加十位。
4. 依次向高位传播。
5. 溢出 9999 后按产品需求回到 0000 或 0001，必须写在规格中。

计数更新建议发生在完整 display frame 之间，避免扫描到一半时四位来自不同时间点。若由 ISR 更新，需双缓冲或原子快照。

## 10. 刷新率和亮度

对四位扫描：

```text
frame_rate = digit_refresh_rate / 4
```

编译 release 要求延时常量绑定 CPU clock/OPTION 并完成 cycle audit。只有用户明确要求后续硬件验收时，才执行测量项：

- 使用示波器测量每位 on-time、off dead-time 和 frame period。
- 确认无肉眼闪烁、相机条纹在产品要求内。
- 峰值段电流、平均占空比符合器件与限流电阻设计。
- 延时常量绑定 CPU clock/OPTION，不跨 profile 照抄。
- 不要仅靠增加 on-time 掩盖段码、位选或驱动能力错误。

## 11. 诊断矩阵

| 实板症状 | 第一优先根因 | 第二优先 | 修复方向 |
|---|---|---|---|
| 上电全亮或多位同时亮 | `PA_PIO=60H` 全关值/混合极性错误 | 切段时 COM 未关闭 | 先验证每个 COM 单独开关 |
| 串影 | 写段码前未全关 | dead-time 太短 | 固化 all-off -> segment -> on |
| `1234 -> 3412` | 物理位序是 COM2,COM3,COM0,COM1 | 业务数组顺序 | 改 COM 映射，不先改段码 |
| 某两位反相/乱码 | 共阳共阴表混用 | DP bit 极性 | 两套编码表分别调用 |
| `0000 -> 0101 -> 0202` | persistent 与 scratch 地址复用 | `SE` 前未重载 A | 审计 SRAM 表和 conversion clobbers |
| 数字偶发跳变 | 扫描与计数并发更新 | ISR 未保护 | frame 边界更新/快照 |
| 亮度不均 | 每位延时或调用路径不同 | 极性驱动能力差异 | 测每位 duty，等化路径 |
| 只显示一位 | 其他 COM OE/PIO 错 | scan loop 未返回 | 单步逐位 probe |

## 12. 最小实板验证顺序（仅后续硬件阶段）

本节只在用户明确要求烧录和实板验收时执行，不是普通编译 release 的前置条件。

1. 全部 COM 关闭，确认无灯。
2. 段线固定显示 `8`，逐个只打开 COM0..COM3，记录极性和物理位置。
3. 分别测试 A..G、DP，确认 PB bit mapping。
4. 固定显示非对称 `1234`。
5. 连续扫描 `8888`，观察串影和亮度。
6. 显示 `0123`、`4567`、`8901`，覆盖全部段码。
7. 最后启用动态计数，记录进位边界 `0009/0010`、`0099/0100`、`0999/1000`、`9999/wrap`。
